# AI 早报 skill · 设计规格

> 公众号：Rich is Good ｜ 每日 AI 早报简讯 ｜ 近 24h 国际国内 AI 资讯

---

## 一、定位

- **内容**：每日 AI 早报，≤10 条精选资讯
- **形态**：纯资讯速递，不加编辑点评
- **目标**：扫读 3 分钟，每天稳定输出
- **公众号品牌**：Rich is Good

## 二、skill 职责边界

**排版 + 轻整理**（不含采集）。AI 的"轻整理"只做 5 件事，不越界：

1. 分类（国内/国际）
2. 排序（时效/重要性）
3. 去重（同一新闻多源合并）
4. 摘要压缩（把导出的冗长描述压成 ≤30 字一句话，不改写实质）
5. 选内容速览条目（选出当天最重要的 3-5 条放进速览区）

**不写点评、不解读、不创作。**

## 三、输入

半结构化导出（RSS/工具导出）→ 归一化成**标准简讯条目**：

```
{ 标题, 一句话摘要(≤30字), 来源媒体, 原始链接, 分类(国内/国际) }
```

## 四、外标题格式（公众号后台设置）

```
AI 早报｜<今日要点新闻>
```

用 `｜` 分隔品牌和要点。示例：`AI 早报｜DeepSeek 自研 AI 芯片，三星利润暴增 1810%`

## 七、关键设计决策（讨论中确定）

| 决策点 | 结论 |
|-------|------|
| 分类体系 | 地域二分：国内/国际，下不细分 |
| 条目规模 | ≤10 条，精选型 |
| 不设传统头条 | 所有条目平铺统一样式 |
| 报头要点 | 外标题承担品牌+要点，正文不重复放大 |
| 内容速览 | 保留，列 3-5 条重点资讯（从当日条目中精选） |
| 相对时间 | **去掉**（节省计算消耗） |
| 点评 | **纯资讯，不加点评** |
| 期数管理 | 只标日期（不要期数编号） |

## 八、待确认的关键问题

### 1. 视觉升级方向
现有是白底+紫色朴素排版。用户之前选了"科技早报"（深色+数据感），但现在看到现有风格后需要确认：

- **方案 A**：保持白底+紫色，优化排版（更精致的间距/字体/组件）—— 最小改动，最稳定
- **方案 B**：白底+紫色保留，但加入科技元素（等宽字体做编号、数据徽章、更精致的卡片）—— 轻度升级
- **方案 C**：改成深色报头+科技青色（v1/v2 预览的方向）—— 视觉差异大

### 2. 内容速览和详情的关系
现有文章速览列 4 条，详情里也有这 4 条（重复标题）。速览是"目录"功能（方便扫读选感兴趣的），还是"摘要"功能（速览是浓缩版，详情是展开版）？

### 3. 来源标注方式
现有用 `📎 信息来源：[1]`（带引用链接）。skill 里的"轻整理"要不要保留这种链接引用？还是简化成媒体名（如"量子位"）？
