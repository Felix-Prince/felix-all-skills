# GZH Formatter — AI 早报公众号文章生成

将采集到的原始资讯数据，处理为可直接发布到微信公众号的 HTML 文章。

## 核心能力

| 能力 | 说明 |
|------|------|
| **多格式输入** | 支持 JSON / Google News 文本 / 自由粘贴 三种格式 |
| **国内/国外分类** | 自动判断新闻来源归属，分两个板块展示 |
| **双语处理** | 英文资讯保留原文 + 自动中文翻译 |
| **7 天主题** | 按星期自动匹配不同配色，每天新鲜感 |
| **纯事实简报** | 每条 1-2 句纯事实陈述，不做任何点评 |
| **微信兼容 HTML** | 输出 inline-style HTML，可直接粘贴到微信公众号编辑器 |

---

## 使用方式

```text
帮我用 gzh-formatter 处理今天的采集数据
```

或者指定文件：

```text
用 gzh-formatter 格式化 data.json 这个文件
```

如果未指定文件，默认读取 `gzh-formatter-skill/data.json`（如果存在）。

你也可以直接粘贴内容：

```text
用 gzh-formatter 处理这些：
CNBC
More
Godfather of AI blasts Musk's xAI as 'failure'...
...
```

## 工作流程

### 第一步：确定日期

- 如果数据中有 `time_str` 字段 → 以数据日期为基准
- 否则以当前系统日期为准
- 日期用于确定星期 → 选择主题

### 第二步：确定主题

根据日期计算星期几，匹配对应的主题色：

| 星期 | 主题 | 板块色 | 气质 |
|------|------|--------|------|
| 周一 | 极简科技 | `#2563EB` | 干净 · 专业 · 冷静 |
| 周二 | 暖心橙 | `#EA580C` | 亲切 · 有温度 · 早晨感 |
| 周三 | 薰衣草 | `#7C3AED` | 优雅 · 清新 · 柔和 |
| 周四 | 极光绿 | `#059669` | 活力 · 生长感 · 科技自然 |
| 周五 | 中国红 | `#DC2626` | 热烈 · 有张力 · 视觉冲击 |
| 周六 | 莫兰迪 | `#9D6B7E` | 高级 · 安静 · 文艺感 |
| 周日 | 深海蓝 | `#1E40AF` | 稳重 · 高端 · 专业深度 |

**完整色值配置见 `references/writing-style.md`。**

### 第三步：读取并解析输入

#### 格式 A：结构化 JSON

```json
[
  {
    "title": "Midjourney Medical: AI-Powered Full-Body Ultrasound Scanner",
    "url": "https://www.engadget.com/...",
    "score": 1277,
    "time_str": "2026-06-18 09:23:15"
  }
]
```

解析规则：
- `title` → 标题
- `url` → 来源链接
- `score` → 排序依据（有则用）
- `time_str` → 日期，也可辅助判断国内/国外（配合域名和标题语言）

#### 格式 B：Google News 文本

```
CNBC
More
Godfather of AI blasts Musk's xAI as 'failure,' says labs are risking a 'big bubble explosion'
19 hours ago
By Arjun Kharpal

The Atlantic
More
America Is Headed Toward the Infinite Workweek
14 hours ago
By Lila Shroff
```

解析规则（段落模式）：
1. 第一行 → 来源名
2. 第二行 → 忽略（"More" 或空行）
3. 第三行 → 标题
4. 第四行 → 相对时间
5. 第五行 → 作者（可选，以 "By " 开头）
6. 空行分隔不同条目

注意：Google News 文本无 URL 和 score，排序按出现顺序。

#### 格式 C：自由粘贴文本

启发式解析：
- 包含 `http://` 或 `https://` 的行 → 提取为 URL
- `标题：` / `##` / `###` 开头 → 视为标题
- 纯文本行 → 合并为正文
- 首行可能是来源名

#### 格式检测优先级

```
检测逻辑：
  1. 内容以 [ 开头且可解析为 JSON → 格式 A
  2. 内容包含 "More\n" 行 + 相对时间模式（N hours/days ago）→ 格式 B
  3. 其他 → 格式 C
```

### 第四步：国内/国外分类

#### 分类策略（按优先级排序）

| 优先级 | 判定依据 | 示例 |
|--------|---------|------|
| 1 | URL 域名含 `.cn` | `xxx.cn/...` → 国内 |
| 2 | URL 域名包含中文拼音或中文平台 | `36kr.com` → 国内 |
| 3 | 标题含中文 | 有中文字符 → 国内 |
| 4 | 其余全部 → 国外 | `.com` / `.io` / 纯英文 → 国外 |

#### 兜底

如果全部是国内或全部是国外 → 只显示一个板块，不保留空板块占位。

### 第五步：排序与精选

#### 排序逻辑

```
有 score 字段 → 按 score 降序排列
无 score 字段 → 保持原始出现顺序
```

#### 精选规则

| 条件 | 处理 |
|------|------|
| 总数 ≤ 10 条 | 全部保留 |
| 总数 > 10 条 | 取前 10 条，尽量保持国内/国外比例均衡 |
| 某板块为 0 条 | 不显示该板块标题 |

**精选策略（超过 10 条时）：**

```
1. 先按顺序从国内取 top，按顺序从国外取 top
2. 确保国内/国外各自至少保留 2 条（如果各自有那么多）
3. 如果某个来源有重复（同一家媒体多条），最多保留 2 条
4. 合并后取前 10 条
```

### 第六步：内容生成

#### 内容速览生成（位于文章开头）

在所有具体新闻之前，生成 3-4 条**凝练概括**，让读者快速了解今天的内容概貌。

**规则：**
- 每条 15-25 字，名词性短语或短句
- 按**主题归纳**，不是按顺序罗列前几条
- 覆盖当天最核心的 3-4 个话题
- 不写完整句子，高度凝练

**示例：**
```
📋 内容速览
  · AI 教父 LeCun 炮轰 xAI 为"失败品"，警示行业泡沫风险
  · 亚马逊自研芯片挑战英伟达，内部调查 AI 数据中心扩张争议
  · Anthropic 连发两条动态：Claude Design 大更新 + 机器人操控突破
  · 美国政策层面：Sanders 提 7 万亿 AI 公有化计划
```

#### 每条正文写作规则

```
中文内容 → 1-2 句提炼核心事实

英文内容 → 两段式：
  段落 1：中文翻译（纯事实，不点评，不补充背景）
  段落 2：英文原文（保持原样不做修改，从原标题和正文中提取 1-2 句）
```

#### 写作范例

```
中文内容：
「百度发布面向开发者的 AI 搜索增强工具"寻知"，支持实时联网检索与多轮对话。」

英文内容：
「Midjourney 宣布进军医疗影像领域，推出基于 AI 的全身超声扫描方案，可辅助医生快速完成全身筛查。
Midjourney has announced its entry into medical imaging with an AI-powered full-body ultrasound scanning solution.」
```

#### 标题翻译规则

- 只翻译第 1 条（头条）的标题，用于公众号外标题（聊天列表显示）
- 其余条目标题：中文保留原文，英文保留英文（正文中加中文翻译）

### 第七步：生成标题

遵循 `references/title-formulas.md` 中的规则生成标题。

### 第八步：生成内容速览

对精选后的全部内容做一次横向扫描，生成 3-4 条主题归纳（按 Writing Style 规范）。

### 第九步：组装 HTML

1. 加载当天主题的色值（从第二步确定）
2. 按 `references/writing-style.md` 的 HTML 模板组装（**不含 h1 主标题**，替换为内容速览）
3. 所有样式使用 inline-style（微信兼容）
4. 封面图使用占位 `<img>` 标签，src 留 `640.jpeg` 的路径（用户需上传到微信并替换）
5. 文末来源标注保持编号一致

### 第十步：输出

**写入文件：**

```
~/.hermes/cron/output/YYYYMMDD_AI早报.html
```

注意：cron 任务环境下，文件会由系统自动写入到 ~/.hermes/cron/output/{job_id}/ 目录。

**如果输出文件已存在则覆盖。**

**告知用户：**

```
✅ AI 早报已生成 →
  文件：{绝对路径}
  主题：{主题名}（{气质描述}）
  共 {N} 条：国内 {X} 条 / 国外 {Y} 条
  封面图：请将 640.jpeg 上传至微信并替换 HTML 中的 src
```

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 数据为空 | 提示"无有效数据"，不生成文件 |
| 只有 1 条 | 正常生成，标题按头条规则处理 |
| 全部国内 / 全部国外 | 只显示一个板块 |
| 头条标题超长（>30 字） | 截断加 `...` |
| JSON 缺少 url 字段 | 来源标注为`[N] 未知来源` |
| 中文媒体是英文标题（如 36kr 的英文报道） | 按域名判为国内，标题保留原文 + 正文加中文翻译 |
| Google News 文本无法解析 | 逐行识别，尽量提取，无法提取的条目跳过并提示跳过了几条 |
