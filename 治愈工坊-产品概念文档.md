# 治愈工坊（HealCraft）—— 一站式治愈系图片内容创作与售卖平台

> 产品概念文档 | 2026-04-24

---

## 一、产品概述

### 1.1 一句话定义

**治愈工坊**是一个治愈系图片内容生产工作站，帮助个人创作者以最低门槛、最高效率完成选题→生图→选图→打包→内容生成→上架素材的全链路内容生产。平台专注于"生产内容"，不碰"分发内容"——所有涉及第三方平台发布、API调用、浏览器自动化的高风险操作一律不做，改为生成可直接复制粘贴的成品内容素材，由用户手动发布到各平台。

### 1.2 核心价值主张

| 痛点 | 现状 | 治愈工坊的解法 |
|------|------|----------------|
| 选题靠灵感 | 创作者凭感觉选题，踩不准热点 | 基于热点数据 + 情绪趋势分析，推荐高潜力选题 |
| 生图没风格 | AI生图风格混乱，批次间不一致 | 风格引擎管理治愈系模板库，保证批次风格统一 |
| 选图凭感觉 | 没有量化标准，选出的图质量不稳定 | 5维评分体系量化选图，AI评分+人工确认 |
| 打包上传繁琐 | 手动压缩、打包、上传网盘，步骤重复 | 自动按规格打包（原图/推文/预览），一键导出到本地 |
| 推文逐平台适配 | 同一内容要反复改格式、改尺寸、改文案 | 一份选题，自动生成多平台适配内容（文案+图片），一键复制 |
| 上架手动写文案 | 闲鱼上架耗时，文案定价没策略 | 自动生成上架文案+定价建议+关键词，一键复制 |

### 1.3 目标用户画像

**主要用户**：个人创作者 / 副业人群

- **画像**：25-35岁，有审美sense，想做副业但没有系统方法论
- **现状**：偶尔在小红书发治愈图，偶尔在闲鱼卖图片包，但不成体系
- **需求**：把"兴趣变现"这件事变得可复制、可量化、可规模化
- **心理**：想要"被治愈"同时也"治愈别人"，获得情感价值和经济回报

**次要用户**：小团队 / 内容工作室

- **画像**：2-3人团队，批量生产治愈系内容
- **需求**：流程标准化、效率最大化、数据驱动决策

## 二、产品架构

### 2.1 流程架构（Pipeline）

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  选题器   │ →  │  生图器   │ →  │  选图器   │ →  │ 打包导出器│ →  │ 内容生成器│ →  │ 上架素材器│
│TopicSelector│ │ImageGenerator│ │ImageCurator │ │ PackExporter │ │ContentGenerator│ │ListingGenerator│
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     ↑               ↑               ↑               ↑               ↑               ↑
     │               │               │               │               │               │
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│                            智能引擎层（Intelligence）                                       │
│  TrendAnalyzer │ StyleEngine │ QualityScorer                                              │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
     ↑               ↑               ↑               ↑               ↑               ↑
     │               │               │               │               │               │
```

> **设计原则**：平台止步于"内容生产"，不碰"内容分发"。所有输出均为可直接复制粘贴的成品素材，用户自行手动发布到各平台，零风控风险。

### 2.2 三层架构说明

| 层级 | 名称 | 职责 | 核心模块 |
|------|------|------|---------|
| Layer 1 | 核心流程 | 用户直接操作的6步内容生产Pipeline | 选题器、生图器、选图器、打包导出器、内容生成器、上架素材器 |
| Layer 2 | 智能引擎 | 为流程提供AI辅助决策 | 热点分析引擎、风格引擎、质量评分引擎 |
| Layer 3 | 内容资产库 | 存储和管理所有内容资产 | 选题库、风格库、图片库、内容模板库、上架模板库 |

## 三、六大核心模块详解

#### 功能定义

基于热点数据和情绪趋势，推荐高商业潜力的治愈系图片选题方向。

#### 选题数据来源

| 来源 | 数据类型 | 采集方式 | 价值 |
|------|---------|---------|------|
| 小红书热门 | 搜索趋势、热门笔记标签 | 手动观察 + 内置热点日历提示 | 最直接的目标市场热点 |
| 微博热搜 | 热搜词条、情感类话题 | 微博开放平台API | 大众情绪风向标 |
| 抖音热门 | 热门BGM、治愈类视频 | 手动观察 | 年轻群体的情绪偏好 |
| 季节气节 | 节日、季节变换、节气 | 内置日历 | 可预测的周期性需求 |
| 天气数据 | 城市天气、极端天气事件 | 天气API | 雨天/雪天/晴天等场景选题 |
| 社会事件 | 大事件后的情绪需求 | 人工标注 | 紧急情绪疗愈需求 |

#### 选题颗粒度设计

选题不是简单的"关键词"，而是包含多维度信息的**选题卡片**：

```typescript
interface TopicCard {
  id: string
  // 主题信息
  theme: string              // 主题名称，如"雨天窗台的猫"
  emotionTag: string         // 情绪标签，如"孤独中的温暖"
  sceneTag: string           // 场景标签，如"室内·窗边"
  seasonTag: string          // 季节标签，如"春雨"
  // 数据支撑
  heatScore: number          // 热度评分 0-100
  searchVolume: number       // 近7天搜索量
  competitorCount: number    // 同类商品数量（闲鱼）
  estimatedDemand: number    // 预估需求量
  // 推荐策略
  suggestedStyleId: string   // 推荐的风格模板ID
  suggestedColorPalette: string[] // 推荐的色彩方案
  suggestedPricingRange: [number, number] // 推荐定价区间
  // 元信息
  source: string             // 数据来源
  createdAt: Date
  expireAt: Date             // 选题有效期（热点有衰减周期）
}
```

#### 选题推荐逻辑

```
热点数据采集 → 情绪趋势分析 → 治愈系相关性过滤 → 商业潜力评分 → 排序推荐
                                                                  ↓
                                                     推荐Top 5选题卡片给用户
```

**情绪趋势分析**的关键洞察：治愈系内容的需求不是恒定的，而是在社会情绪低谷时爆发（如考试季、加班季、换季、疫情后）。选题器应该能捕捉这些情绪波动。

#### 选题类型分类

| 类型 | 示例 | 特点 | 热度周期 |
|------|------|------|---------|
| **日常治愈** | 咖啡与书、窗台植物 | 长期稳定需求 | 持续性 |
| **季节治愈** | 春日樱花、冬日暖炉 | 周期性爆发 | 季节性 |
| **情绪治愈** | 考试加油、深夜emo | 紧急需求 | 事件驱动 |
| **社交治愈** | 生日祝福、情侣纪念 | 礼赠场景 | 节日驱动 |
| **审美治愈** | 极简空间、色彩渐变 | 收藏需求 | 持续性 |

### 3.2 生图器（ImageGenerator）

#### 功能定义

基于选题卡片，批量生成风格统一的治愈系图片（20-30张），确保批次内风格一致性。

#### 生图策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 生成数量 | 25张 | 在20-30之间取中间值，兼顾质量和数量 |
| 风格模板 | 由选题卡片推荐 | 通过风格引擎关联 |
| 尺寸规格 | 2048×1536（4:3横图）+ 1536×2048（3:4竖图） | 同时生成横竖两种比例 |
| 批次种子 | 同批次共享seed前缀 | 保证风格一致性 |
| 质量优先级 | quality > speed | 宁可慢一点，质量要过关 |

#### 风格引擎（StyleEngine）

风格引擎是解决"批量生图风格混乱"的核心：

```typescript
interface StyleTemplate {
  id: string
  name: string                    // 如"水彩治愈风"、"摄影写实治愈风"
  basePrompt: string              // 基础prompt模板
  negativePrompt: string          // 负面prompt（排除不想要的元素）
  loraId?: string                 // LoRA模型ID（如果有定制训练）
  seedPrefix?: string             // 种子前缀（控制风格方向）
  colorPaletteConstraint: string  // 色彩约束指令
  compositionRules: string[]      // 构图规则指令
  excludedElements: string[]      // 排除的元素（如：文字、人脸、现代建筑）
  qualityThreshold: number        // 该风格的最低质量阈值
  examples: string[]              // 示例图片URL（风格参考）
}
```

#### 治愈系风格分类

| 风格 | 描述 | 目标场景 | 推荐AI模型 |
|------|------|---------|-----------|
| **水彩治愈风** | 柔和水彩笔触，暖色调 | 壁纸、明信片 | MidJourney |
| **摄影写实治愈风** | 拍照级真实感，自然光 | 收藏、头像 | MidJourney / SDXL |
| **插画治愈风** | 清新插画线条，简约配色 | 小红书推文、公众号插图 | DALL-E / MidJourney |
| **像素治愈风** | 复古像素风格，怀旧感 | 游戏圈、复古爱好者 | SD + 像素LoRA |
| **油画治愈风** | 印象派油画质感，浓郁色彩 | 高端收藏、艺术爱好者 | MidJourney |
| **极简治愈风** | 极简构图，大量留白 | 桌面壁纸、设计素材 | MidJourney |

#### 生图质量保障机制

```
生成图片 → AI痕迹检测 → 美学初筛 → 异常元素检测 → 合格图片入库
   ↓          ↓            ↓            ↓
  异常      明显AI味     低美学分    文字/人脸    → 标记废弃
```

- **AI痕迹检测**：检测手指变形、文字乱码、脸部异常等常见AI生成问题
- **美学初筛**：使用美学评分模型对每张图做初步评分，低于阈值直接废弃
- **异常元素检测**：检测是否有不想要的元素（如：文字、人脸、现代建筑），根据风格模板配置

### 3.3 选图器（ImageCurator）—— 核心创新模块

#### 功能定义

从20-30张生图中，通过5维评分体系 + 人工确认，筛选出约10张高质量、风格统一的治愈系图片。

#### 5维评分体系

这是选图器的核心创新——将"凭感觉选图"变为"有方法论选图"：

| 维度 | 权重 | 评分方法 | 1分示例 | 5分示例 |
|------|------|---------|---------|---------|
| **情绪共鸣度** | 30% | 语义匹配：图片内容与选题情绪标签的匹配度 | 冷冰冰的场景，无治愈感 | 一看就能感受到温暖 |
| **美学质量** | 25% | AI美学评分模型 + 色彩分析 | 色彩杂乱，构图失衡 | 色彩和谐，构图完美 |
| **风格一致性** | 20% | 图片特征向量与批次风格中心的距离 | 风格偏离，与其他图格格不入 | 风格统一，视觉连贯 |
| **商业潜力** | 15% | 历史成交数据相似度 + 场景适用度 | 小众场景，无明显购买动机 | 广泛适用，有明确购买场景 |
| **技术合规性** | 10% | AI痕迹检测 + 尺寸规格校验 | 明显AI瑕疵，尺寸不符 | 无AI痕迹，规格完美 |

#### 选图流程

```
输入：25张生图 + 选题卡片 + 风格模板
        ↓
Step 1: 技术合规性过滤（权重10%）
   → 去除有明显AI瑕疵、尺寸不符的图片
   → 预估废弃 3-5张
        ↓
Step 2: 5维综合评分
   → 每张图计算：综合分 = 0.30×情绪 + 0.25×美学 + 0.20×风格 + 0.15×商业 + 0.10×技术
        ↓
Step 3: 风格一致性聚类
   → 将高分图片按风格特征聚类
   → 选择最大簇（风格最统一的群体）
   → 预估剩余 12-15张
        ↓
Step 4: 商业潜力排序
   → 在最大簇内按商业潜力排序
   → 取Top 10
        ↓
Step 5: 人工确认
   → 用户查看AI推荐的10张图
   → 可以替换、调整、补充
   → 最终确认10张入选图
        ↓
输出：10张精选图 + 评分报告 + 选图理由
```

#### 选图评分报告

每次选图生成一份评分报告，帮助用户理解AI的推荐逻辑：

```
═══════════════════════════════════════

═══════════════════════════════════════

  综合排名  │  情绪  │  美学  │  风格  │  商业  │  技术
  ─────────┼───────┼───────┼───────┼───────┼──────
  #1 ★★★★★ │ 4.8   │ 4.9   │ 4.7   │ 4.5   │ 5.0
  #2 ★★★★★ │ 4.7   │ 4.6   │ 4.8   │ 4.4   │ 5.0
  #3 ★★★★☆ │ 4.5   │ 4.7   │ 4.6   │ 4.2   │ 4.8

  #10 ★★★☆☆ │ 3.8   │ 4.1   │ 4.0   │ 3.5   │ 4.5
```

### 3.4 打包导出器（PackExporter）

#### 功能定义

将精选图片按多种规格自动打包导出到本地，生成可直接分享的文件包，用户自行上传到网盘或文件服务。

> **设计原则**：平台只负责打包和导出，不接触任何第三方网盘API。用户手动上传至百度网盘、阿里云盘或自建文件服务，规避所有API不可用和风控风险。

#### 功能要点

| 功能 | 说明 |
|------|------|
| **自动打包** | 10张精选图按多种规格自动打包为ZIP，按"主题+日期"命名 |
| **多规格包** | 同时生成：原图包（高清）+ 推文包（压缩）+ 预览包（缩略图） |
| **一键导出** | 点击"导出"按钮，所有规格包自动生成到用户指定的本地目录 |
| **链接管理提示** | 导出后显示上传建议（推荐哪些网盘/文件服务、如何设置分享链接） |
| **分享文案生成** | 自动生成发货用的网盘分享文案，用户上传后填入链接即可使用 |

#### 导出包规格

```
本地导出目录: /{用户指定路径}/[雨天窗台的猫]_2026-04-24/
│
├── 原图包_高清.zip (10张, 2048×1536, PNG)
│   每张约 3-5MB, 总约 30-50MB, 用于售卖发货
│
├── 推文包_社交.zip (10张, 1080×1440, JPG, 80%质量)
│   每张约 200-500KB, 适合小红书/微博/公众号等社交平台发布
│
├── 预览包_缩略图.zip (10张, 400×300, JPG)
│   每张约 30KB, 用于闲鱼上架展示、封面图
│
└── README.txt
    图片说明、使用场景、版权声明
```

### 3.5 内容生成器（ContentGenerator）

#### 功能定义

一份选题，自动生成适配多平台的成品内容（文案 + 图片排版建议 + 标签），用户一键复制后手动发布到各平台。

> **设计原则**：平台只生成内容，不发布内容。所有平台发布操作由用户手动完成，规避API风控、账号封禁等风险。一键复制 = 用户点击"复制"按钮，内容复制到剪贴板，用户打开目标平台粘贴发布。

#### 核心定位

内容生成器不是"自动发文工具"，而是**内容生产引擎**——一次选题，产出N份平台适配内容，用户只需复制粘贴即可发布。

```
一份选题 ──→ 内容生成器 ──→ 小红书成品内容（一键复制）
                         ──→ 公众号成品内容（一键复制）
                         ──→ 微博成品内容（一键复制）
                         ──→ 抖音图文内容（一键复制）
```

#### 生成内容类型

对每个选题，内容生成器产出以下内容包：

| 内容项 | 说明 | 用户操作 |
|--------|------|---------|
| **平台文案** | 每个平台独立的标题+正文+标签 | 点击"复制文案"→ 打开平台粘贴 |
| **图片选择建议** | 推荐每个平台使用哪几张图、排列顺序 | 参考建议，手动选图上传 |
| **图片尺寸适配** | 推文包中已按平台规格压缩好的图片 | 从导出目录直接上传 |
| **发布时间建议** | 该平台最佳发布时间段 | 参考建议，选择时间发布 |

#### 多平台内容适配规范

### 3.6 上架素材生成器（ListingGenerator）

#### 功能定义

自动生成闲鱼上架所需的全部素材（标题、描述、定价建议、关键词、封面图建议），用户一键复制后手动在闲鱼发布。

> **设计原则**：平台只生成上架素材，不碰闲鱼操作。闲鱼无官方API，浏览器自动化有封号风险，唯一安全的方式是"素材一键复制 + 手动上架"。

#### 生成内容清单

| 素材项 | 说明 | 用户操作 |
|--------|------|---------|
| **商品标题** | 闲鱼SEO优化标题，含关键词 | 点击"复制标题" → 闲鱼粘贴 |
| **商品描述** | 完整详情页文案 | 点击"复制描述" → 闲鱼粘贴 |
| **定价建议** | 基于选题热度和市场分析的推荐价格 | 参考定价，手动输入 |
| **关键词标签** | 搜索优化关键词 | 点击"复制关键词" → 闲鱼粘贴 |
| **封面图建议** | 推荐哪张图做封面，预览包中已裁好 | 从预览包选图上传 |
| **自动回复文案** | 买家咨询/下单后的回复模板 | 点击"复制回复" → 闲鱼粘贴 |

#### 定价策略框架

| 商品类型 | 定价区间 | 定价逻辑 |
|---------|---------|---------|
| **基础包**（10张高清原图） | ¥5-15 | 低价引流，吸引新买家 |
| **精选包**（10张+5张变体） | ¥15-25 | 中等定价，主推产品 |
| **主题系列**（3组同主题） | ¥30-50 | 高价值打包，主题收藏者 |
| **定制版**（+1张定制图） | ¥25-40 | 高溢价，个性化需求 |

**动态定价建议因素**：

- 热点效应：选题热度越高，定价可以适当上浮
- 季节因素：季节性选题在对应季节可提价
- 新品效应：新品上架前3天可设低价引流

## 六、风险评估与对策

### 6.1 关键风险矩阵

| 风险 | 严重度 | 可能性 | 对策 |
|------|--------|--------|------|
| **闲鱼虚拟商品政策收紧** | 🔴高 | 🟡中 | 多平台分散售卖 + 实体化商品（明信片、日历）+ 内容付费模式 |
| **AI生图版权争议** | 🟡中 | 🟡中 | 建立私有风格库（LoRA训练）+ 标注AI生成 + 关注政策动态 |
| **同质化竞争** | 🟡中 | 🔴高 | 选题差异化 + 风格差异化 + 选图方法论差异化 |
| **推文平台限流** | 🟡中 | 🟡中 | 内容优先策略 + 多平台分散 + 自然增长不刷量 |
| **AI生图质量不稳定** | 🟡中 | 🟡中 | 风格引擎约束 + AI痕迹检测 + 人工兜底 |
| **LLM生成的文案风格不稳定** | 🟡中 | 🟡中 | 治愈系文案风格模板 + 人审确认 + 模板迭代 |
| **用户手动发布耗时** | 🟡中 | 🟡中 | 一键复制极大减少操作时间（从15分钟降至2-3分钟），ROI仍然可观 |

> **已消除的风险**：网盘API不可用（改为本地打包+手动上传）、闲鱼无API（改为素材一键复制+手动上架）、浏览器自动化封号（不使用任何自动化）、小红书无发布API（不尝试自动发布）、平台风控封号（不触碰任何第三方平台操作）

### 6.2 最关键假设（必须先验证）

**假设1：治愈系图片在闲鱼有足够购买需求**

- 验证方法：先手动上架5-10个图片包，观察7天成交量
- 失败标志：7天内零成交或极低询价
- 失败对策：转向内容付费（小红书专栏、公众号付费阅读）

**假设2：AI评分选出的图比纯人工选的图成交率更高**

- 验证方法：同一主题，AI选10张 vs 人工选10张，分别上架对比成交率
- 失败标志：AI选图成交率显著低于人工选图
- 失败对策：降低AI评分权重，转为"AI辅助排序 + 人工最终决策"

**假设3：推文能有效引流到闲鱼成交**

- 验证方法：发布推文后，闲鱼浏览量和询价量是否有明显增长
- 失败标志：推文发布前后闲鱼数据无变化
- 失败对策：推文策略调整（更软性的购买引导）或转向SEO引流

## 七、技术选型建议

### 7.1 AI生图技术栈

| 技术 | 用途 | 优劣 |
|------|------|------|
| **MidJourney** | 生图核心（MVP） | 质量最高，但无API，需手动操作Discord |

| **DALL-E 3** | 插画风格生图 | prompt理解最强，但成本较高 |

**推荐方案**：
- MVP阶段：MidJourney手动操作（质量优先验证）+ FLUX.1 Dev API（批量自动化）

| **特征向量提取 + 聚类分析** | 风格一致性评分 |
| **AI痕迹检测模型** | 技术合规性评分 |

### 7.3 平台技术栈

| **Redis** | 缓存和任务队列 |

| **剪贴板API** | 一键复制功能（Electron clipboard API） |

## 九、演进路线

### 9.1 Phase 1 —— MVP（最小可行产品）

**目标**：验证核心流程，证明"治愈系图片可以变现"

**范围**：选题→生图→选图→打包导出→内容生成→上架素材（6步核心流程，全部为内容生产）

| 模块 | MVP功能 | 实现方式 |
|------|---------|---------|
| 选题器 | 手动输入主题 + 微博热搜推荐 + 内置热点日历 | 微博API + 手动输入 + 日历 |
| 生图器 | 预设3种风格模板，批量生成25张 | MidJourney手动 + FLUX.1 Dev API |
| 选图器 | 5维AI评分 + 人工确认 | CLIP + NIMA本地部署 |
| 打包导出器 | 自动打包3种规格ZIP到本地目录 | 本地打包引擎 |
| 内容生成器 | 4个平台内容一键复制（小红书/公众号/微博/抖音） | LLM文案生成 + 剪贴板API |
| 上架素材器 | 闲鱼上架素材一键复制（标题/描述/定价/关键词） | 模板引擎 + 剪贴板API |

**MVP验证指标**：

| 指标 | 目标值 | 衡量标准 |
|------|--------|---------|
| 首次上架成交 | 7天内首次成交 | 证明产品可变现 |
| 单选题耗时 | <30分钟（含手动发布） | 证明流程效率 |
| 生图通过率 | >60% | 证明生图质量可控 |
| 选图人工调整 | <3张替换 | 证明选图评分有效 |
| 内容生成满意度 | 用户认为80%以上文案可直接使用 | 证明内容生成器价值 |

**MVP时间预估**：4-6周
