SECTION 01

系统定位与核心理念

1.1 盾与矛的选择

❌ 错误定位：矛

公告出来 → 告诉你"买入机会" → 你去追高 → 接盘侠

信息永远滞后于内幕，散户用"买入信号"永远慢一拍

✅ 正确定位：盾

信息出来 → 评估"预期差多少"+"持仓风险多大"+"信号是否共振" → 防守决策

在公开信息中提取更高维度的认知，而非抢先获取信息

核心洞察：AI 的价值不是"抢先获取信息"（你永远抢不过内幕），而是"在公开信息中看到别人看不到的东西"。防守优先、预期差量化、信号共振。

1.2 核心价值三层模型

🎯 第一层：预期差量化

不是告诉你"公告说增长50%"（谁都知道），而是告诉你市场只预期20%，公告50%，预期差+30%

超预期个股公告后5-10日仍有正向超额收益

核心 alpha
🛡️ 第二层：持仓风险预警

不帮你"找买入机会"，帮你守住已有利润

你已有持仓 → 有明确场景 → 风险评估 → 减仓/止盈提醒

最易落地
🔗 第三层：信号共振检测

单个信号确实滞后，但多个信号在短时间内集中出现，往往意味着趋势还没被充分定价

对抗滞后
SECTION 02

空间策略：热点板块聚焦

2.1 为什么不全市场扫描？

现实考量：A股5000+股票全扫描 = 高成本 + 低信噪比。A股是情绪市场，冷门板块的公告没人看，热点板块的公告被放大反应。同样的"业绩预增50%"，在冷门板块涨2%就结束，在热点板块可能直接涨停。

2.2 动态板块池设计

全市场板块池（动态筛选，每周更新） │ ├─ 热度量化评分 = f(近5日涨幅, 涨停密度, 换手率偏离, 资金集中度) │ ├─ Top 5-8 板块 → 进入「重点监控池」（约 50-150 只股票） │ ├─ 当前热门板块（如：AI算力、低空经济） │ └─ 曾经热门+可能回流的板块（如：新能源、半导体） │ ├─ 衰减期板块 → 进入「观察池」（再跟踪 1-2 周） │ └─ 冷门板块 → 不跟踪，省成本

2.3 板块热度量化评分

维度 计算方式 权重 数据源
近涨幅 板块近5日累计涨跌幅 30% AKShare
涨停密度 板块内涨停家数 / 板块总家数 25% AKShare

3.3 无公告日的典型场景

⚠️ 场景A：情绪退潮

触发条件：

• 涨停家数：8→2（情绪降温）

• 换手率：5.2%→3.8%（资金撤退）

• 龙头高开低走（疲态）

• 北向净流出2.3亿

风险评分：7/10 → 建议减仓1/3

⚡ 场景B：关联传导

触发条件：

• 同板块B票发减持公告

• 另2只票龙虎榜机构净卖出

• 板块跌1.5%（大盘仅0.3%）

风险评分：6/10 → 关注关联风险

✅ 场景C：一切正常

触发条件：

• 板块涨1.2%（大盘0.3%）

• 北向净流入1.8亿

• 龙头稳步上涨，量价配合

• 均在20日线上方

风险评分：2/10 → 安心持有

盾的价值不只是"告诉你危险"，也包括"告诉你安全"——减少焦虑性卖出，也是一种防守。

关键洞察：板块从"加速"进入"分化"是整个系统最重要的防守信号之一——而这个判断完全不需要任何公告，仅依赖日频行情数据即可量化。

4.2 分化判断五大量化指标

指标一：板块内涨幅离散度 权重 30% · 最核心

计算方式：板块内所有个股当日涨幅的标准差

阶段龙头跟风后排标准差
加速期+6%+4~5%+2~3%小（~1.5%）
分化期+10%+1%-3%大（~4.2%）

触发条件：离散度突破近5日均值 + 2倍标准差（动态阈值，详见第5章）

指标二：涨停家数占比 权重 20%

计算方式：板块内涨停家数 / 板块总家数

阶段涨停占比解读
加速期20%+（40只票8只涨停）资金共识强
分化期<5%（只剩1-2只涨停）只剩龙头撑场
指标三：板块内涨跌比 权重 20%

计算方式：板块内上涨家数 : 下跌家数

阶段涨跌比解读
加速期8:2 以上多数票在涨
分化期5:5 甚至更低板块已分裂
指标四：龙头偏离度 权重 15%

计算方式：龙头股涨幅 - 板块平均涨幅

阶段偏离度解读
加速期龙头多涨3%（合理领先）龙头带着板块走
分化期龙头多涨8%+龙头独立走妖，板块已脱节
指标五：量价背离 权重 15%

计算方式：板块指数涨幅 vs 板块总成交额变化

阶段指数成交额解读
加速期涨3%增40%量价齐升，健康
分化期涨1%降20%量价背离，危险

SECTION 05

动态阈值方法论

核心原则：指标阈值不凭空设想，不使用固定数字。用历史百分位法动态计算，并按市场状态（牛/熊/震荡）分层自适应。

5.1 市场状态客观分类

不主观判断"我觉得是牛市"，用四个量化指标自动分类：

维度计算方式牛市态熊市态震荡态
趋势方向 上证收盘 vs 20日均线 价格 > 均线+2% 价格 < 均线-2% 其他
市场广度 全A上涨家数占比20日均值 > 60% < 40% 40%-60%
成交活跃度 全A成交额 / 过去250日均值 > 1.5倍 < 0.7倍 0.7-1.5倍
波动率 上证20日收益率标准差 正常波动 高波动(>P80)或低波动( 正常
自动判定规则： 趋势↑ + 广度宽 + 活跃 → 牛市态 趋势↓ + 广度窄 + 萎缩 → 熊市态 其他组合 → 震荡态（细分强震荡/弱震荡）

5.2 历史百分位法

在每种市场状态下，统计各指标的历史分布，用百分位作为动态阈值：

示例：板块涨幅离散度的历史分布

过去5年所有交易日（约1200天），按市场状态分层：

市场状态天数P25P50P75P90（触发线）P95
牛市态 300天 1.8%2.5%3.2% 4.1%5.0%
熊市态 250天 2.1%2.8%3.5% 4.3%5.2%
震荡态 650天 1.5%2.0%2.8% 3.6%4.5%

⚡ 注意：牛市离散度整体偏高，4%在牛市是常态，在震荡态是极端值——固定阈值不可行！

5.3 每日计算流程

1. 判断今日市场状态（4指标自动分类） 2. 对每个跟踪板块，计算5个量化指标的当前值 3. 对照当前市场状态下的历史分布，计算每个指标的百分位 4. 百分位 > P85 → 该指标触发 5. 按权重汇总为"分化评分" 6. 评分 ≥ 70 → 板块进入分化期 → 防守信号触发

5.4 验证闭环

🔄 阈值不是"定了就完"，必须有验证闭环
1. 用历史5年数据计算所有交易日的分化评分 2. 标记所有评分≥70的日期 3. 回测：这些日期之后3日/5日板块超额收益是否显著为负？ 4. 如果不显著 → 调整触发百分位（如 P85→P90） 5. 如果显著 → 当前设定有效 6. 每季度重新统计+验证，阈值随市场结构变化而进化 预期目标： - 分化信号触发后3日，板块超额收益为负的概率 > 60% - 信号触发准确率（真正分化的占比）> 50% - 作为防守信号：宁可多预警，不可漏预警
SECTION 06

预期差分析体系

预期差是"盾"最锋利的部分——不是公告数字本身，而是公告与市场预期的偏差。

6.1 完整版 vs MVP降级方案

方案对比基准精度数据可用性成本
完整版 公告值 vs 券商一致预期 最高 ❌ 需 Wind/iFinD 几万/年
方案A 公告值 vs 行业均值 中等 ⚠️ AKShare可算 免费
方案B 公告值 vs 公司历史趋势 中等 ✅ AKShare完整 免费
方案C 公告值 vs 股价隐含预期 较高 ✅ AKShare完整 免费

6.2 MVP推荐组合：方案B + 方案C

方案B：历史自身趋势对比

逻辑：公告增速50% vs 该公司过去3年平均增速20% → 自身差+30%

历史数据 AKShare 完整覆盖

免费可用
方案C：股价隐含预期反推 ⭐推荐

逻辑：如果股价在公告前已涨30%，市场隐含预期已经很高

公告增速50%但市场可能已预期60% → 实际低于预期

用公告前20日涨幅作为"市场隐含预期"的代理变量

最巧妙·免费可用
方案C 的核心价值：不需要外部预期数据，从股价本身反推市场隐含预期——这是"用市场自己的语言来判断市场预期"的思路，数据完全免费可用。

SECTION 08

运行机制设计

8.1 为什么不是24h实时监控？

❌ 24h实时监控的问题

• 成本高：服务器+数据流+API调用24h运行

• 效率低：A股非交易时段没有行情数据

• 价值低：凌晨3点推送你也不会操作

• 关键信息不需要实时：公告盘后发，外盘影响盘前评估

✅ 定时驱动 + 事件触发的优势

• 成本可控：只在关键时间节点运行

• 信号精选：不是什么垃圾信息都推

• 用户友好：在用户最需要决策的时刻推送

• 事件触发保证重大事件不遗漏

SECTION 11

风险与限制

⚠️ 合规红线

系统定位为"信息辅助工具"，而非"投资建议"——后者需要投顾牌照。所有推送内容应使用"建议关注/评估/考虑"等措辞，避免"建议买入/卖出"等确定性指令。

⚠️ LLM 数字幻觉

大模型对数字敏感度不够，关键数值（增速百分比、资金净额）必须结构化验证，不能纯靠 LLM 输出。所有关键数字应从数据源直接提取，LLM 只负责"解读"而非"计算"。

⚠️ 数据延迟

免费数据源通常15分钟延迟，对短线交易不够用。系统定位为"日频防守"而非"分钟级交易"，延迟影响可控。

⚠️ 历史回测 ≠ 未来

历史回测表现不代表未来收益。所有基于历史统计的阈值和评分都有失效的可能。务必在推送中加免责声明。

📌 数据爬取合规

巨潮资讯和东方财富网页爬取处于合规灰色地带。建议控制爬取频率（非高峰期），仅用于个人研究，不做商业分发。
