# 照片初筛工具 — 实现路线规划

> 文档版本: v1.0
> 创建日期: 2026-06-08
> 关联文档: 照片筛选工具-设计规划文档.md
> 状态: 规划阶段

---

## 核心原则

> **每个阶段结束后，都能拿到一个能用的工具。不是半成品，而是可用的产品——只是功能逐步增强。**

---

## 阶段总览

| 阶段 | 名称 | 开发耗时 | 累计 | 里程碑 |
|------|------|---------|------|--------|
| 零 | 项目骨架 | 1h | 1h | 能跑、能看图 |
| 一 | 手动快筛 | 2h | 3h | 第一个可用版本，键盘快筛 |
| 二 | 技术初筛 | 1h | 4h | 硬伤自动淘汰 + 技术评分 |
| 三 | AI 审美评估 | 3.5h | 7.5h | 核心能力上线，智能筛选 |
| 四 | 智能分组 | 1h | 8.5h | 连拍一键选优 |
| 五 | 学习个性化 | 1h（实现） | 9.5h | 越用越准（长期隐性能力） |

**纯开发耗时约 10 小时。** 真正的时间投入在 prompt 调优和长期学习积累上。

---

## 阶段零：项目骨架（1h）

### 产出

一个能打开文件夹、显示照片缩略图的应用。

### 任务

| 任务 | 说明 |
|------|------|
| 技术选型决策 | Tauri / Electron / 其他，选定开干 |
| 项目脚手架 | 项目结构、基础配置 |
| 文件扫描 + JPG/RAW 配对识别 | 同文件名不同扩展名自动关联 |
| 基础 UI | 窗口 + 缩略图网格展示 |

### 验证点

- 打开一个 800 张照片的文件夹，缩略图加载是否流畅

---

## 阶段一：手动快筛（2h）

### 产出

不接任何 AI，纯键盘操作快速筛选。操作即文件移动，比文件管理器快得多。

这是工具的**底线价值**——即使没有 AI，工具也已经可用。

### 任务

| 任务 | 说明 |
|------|------|
| 子文件夹创建 + 首次启动确认弹窗 | 创建 保留/删除/待定 子文件夹 |
| 键盘操作（P/X/U/星标/方向键） | 核心交互 |
| 文件移动逻辑 | JPG+RAW+XMP 配对整体移动（同磁盘路径重命名） |
| 单张预览 + 临时放大（Z 键） | 按住放大，松开恢复 |
| Ctrl+Z 撤销 | 操作历史栈 |
| 断点续传 | 检测已有子文件夹，恢复上下文 |

### 验证点

- 用真实照片完整走一遍筛选流程，操作是否流畅
- 文件移动是否秒级完成
- 配对移动是否可靠（不会出现留了 JPG 留了 RAW）

### 快捷键

| 动作 | 快捷键 |
|------|--------|
| 保留 | P |
| 删除 | X |
| 待定 | U |
| 星标 1-5 | 1 2 3 4 5 |
| 去除星标 | 0 |
| 下一张 | → / 空格 |
| 上一张 | ← |
| 下一组 | Tab |
| 放大查看 | Z / F |

## 阶段二：技术初筛（1h）

### 产出

传统算法做硬过滤和技术评估，不需要 AI 模型。硬伤照片自动淘汰，技术评分辅助判断。

### 任务

| 任务 | 说明 |
|------|------|
| 硬过滤 | 模糊检测(Laplacian)、纯黑/纯白帧、文件损坏 |
| 基础技术评估 | 曝光分析(直方图)、清晰度评估、EXIF 提取 |
| XMP 读写 | 写入评分和标签，兼容 Lightroom/Capture One |
| 评估结果展示 | 技术档案 + 按技术分排序 |

### 验证点

- 硬过滤误杀率——拿 100 张真实照片跑一遍，好片是否被误判
- 技术评估和直觉是否对得上

---

## 阶段三：AI 审美评估（3.5h）

### 产出

工具从"快筛器"升级为"智能助手"——审美驱动留存价值判断。

**这是核心差异化能力。**

### 3a：Prompt 验证（2.5h）

用 Dify 作为 prompt 实验室，验证后再迁移到工具。

| 任务 | 说明 |
|------|------|
| Dify 部署 + 连接 Ollama | Docker 本地部署，连 gemma3:4b 和 InternVL |
| 场景识别 prompt 调通 | 人像/风景/街拍/活动/建筑 分类 |
| 审美评估 prompt 调通 | **核心耗时项**——反复迭代，找合适的 prompt |
| 深度评估 prompt 调通 | 边界照片的细致评估 |
| 组内对比 prompt 调通 | 两张照片的对比判断 |

**产出：验证过的 prompt 模板 + 模型路由逻辑**

### 3b：集成到工具（1h）

| 任务 | 说明 |
|------|------|
| Ollama API 调用封装 | HTTP 请求 + 结构化输出解析 |
| Prompt 模板配置系统 | YAML 配置文件加载 |
| 轻量预判流程（gemma3:4b） | 全量扫描 + 结果分流 |
| 深度评估流程（InternVL） | 边界照片深度评估 |
| 审美档案 UI 展示 | 四维度分数、核心亮点 |
| 评判语境推断 | RAW→后期意图、场景→评估标准调整 |

### 验证点

- 100 张真实照片：AI 判断和你的直觉一致率多少？
- 800 张全量跑一遍，速度是否可接受？
- "痛苦抉择"照片（高审美 + 致命技术伤）是否正确呈现？

---

## 阶段四：智能分组（1h）

### 产出

相似照片自动分组，每组推荐关键帧，一键选优。

### 任务

| 任务 | 说明 |
|------|------|
| 相似度计算 + 自动分组 | 图像特征 + 时间戳 |
| 关键帧推荐 | 基于审美评分选组内最优 |
| 组内批量操作 UI | "就选推荐帧，其余删除" |
| 包围曝光识别 | EXIF 包围标记检测 → 素材组 |

### 验证点

- 连拍组是否正确识别
- 推荐帧你是否认同

---

## 阶段五：学习个性化（1h 实现，长期隐性能力）

### 产出

工具越用越懂你。开发实现 1h，但学习效果是长期积累的。

### 任务

| 任务 | 说明 |
|------|------|
| 理由选择 UI + 渐退机制 | 删除时弹理由选项，按置信度消退 |
| 决策记录 + 偏差分析 | 存储每次决策，提取偏好模式 |
| 偏好画像展示 | 你的审美偏好摘要 |
| 偏好应用到评估 | 学到的偏好影响后续评分 |
| 遗忘/重置/风格快照 | 偏好衰减和管理 |

### 长期隐性能力

学习机制不是"做完就结束"的功能，而是一个**持续运行、持续生效**的隐性能力：

- 每次使用都在积累偏好数据
- 偏好随使用量增加而收敛（10次→低置信度，30次→中，100次→高）
- 人的审美会变——遗忘机制让系统跟着你变
- 长远可训练个人化轻量模型（当数据积累足够时）

---

## 时间分布的真实情况

```
纯代码开发:    ~10h  (vibe coding 提效显著)
prompt 调优:   不确定 (阶段三3a的核心耗时，取决于模型能力和迭代次数)
长期学习积累:   持续   (阶段五上线后每次使用都在积累)
```

**真正的瓶颈不在写代码，在于：**

1. **Prompt 质量** — 决定审美评估的方向是否对。阶段三 3a 的 2.5h 是预估，实际可能更多也可能更少，取决于模型表现
2. **学习数据积累** — 偏好模型需要真实决策数据喂养，这不是开发时间能加速的，是用出来的

---

## 风险与缓解

| 风险 | 阶段 | 缓解 |
|------|------|------|
| 本地模型审美判断不准 | 三 | Dify 验证阶段暴露问题；prompt 调不好换模型或加云端增强 |
| 800 张照片评估太慢 | 三 | 分级评估 + 工具端并发控制 |
| 缩略图加载卡顿 | 零/一 | 虚拟滚动 + 懒加载 |
| Ollama 服务不稳定 | 三 | 重试机制 + 降级提示（标记为"AI 不可用"） |
| Prompt 调优耗时超预期 | 三 3a | 先用简单 prompt 上线，后续持续迭代优化 |
