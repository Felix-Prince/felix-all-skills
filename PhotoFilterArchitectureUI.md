## 一、技术选型

### 1.1 桌面应用框架：Tauri

| 维度 | 选择 | 说明 |
|------|------|------|
| 框架 | Tauri | 包体小(~10MB)、性能好、Rust 后端文件操作自然 |
| 后端语言 | Rust | 直接调 OS API，文件移动/扫描/XMP 读写 |
| 前端框架 | Vue 3 | 上手简单、中文生态好、Tauri 官方支持 |
| 前端构建 | Vite | 快，Tauri + Vue 模板默认集成 |

**切换预案**：阶段一验证时，如 Tauri 实现不如预期（Rust 编译慢、前端通信复杂等），可切换到 Electron。前端 Vue 代码可复用，只换后端壳子。

### 1.2 模型通信

| 组件 | 方式 | 地址 |
|------|------|------|
| 审美评估模型 | HTTP REST API | Ollama localhost:11434 |
| 图片传递 | base64 编码 | 嵌入 API 请求体 |
| 输出解析 | 结构化 JSON | prompt 中要求模型输出 JSON 格式 |

### 1.3 数据存储

| 数据 | 存储方式 | 路径/说明 |
|------|---------|----------|
| 照片分类状态 | 文件夹位置 | 照片在 保留/删除/待定/ 哪个文件夹就是状态 |
| 审美档案 + 技术档案 | XMP sidecar | 跟照片走，Lightroom 可读 |
| 用户偏好 | 本地 JSON 文件 | 工具配置目录下 (~/.snappick/preferences.json) |
| 决策历史 | 本地 SQLite | 学习机制的结构化数据 |
| Prompt 模板 | YAML 配置文件 | prompts/ 目录下，按场景分类 |
| 会话进度 | .session 文件 | 工作目录下，断点续传用 |
| 应用配置 | 本地 JSON 文件 | 工具配置目录下 (~/.snappick/config.json) |

### 3.2 三区域说明

| 区域 | 位置 | 大小 | 内容 |
|------|------|------|------|
| **保留区** | 上方 | 占主体面积的 ~70% | 系统建议保留 + 你确认保留的照片 |
| **待定区** | 下方左侧 | 下方区域的一半 | 犹豫中的照片 |
| **删除区** | 下方右侧 | 下方区域的一半 | 直接 pass 掉的照片 |

- 每个区域有独立的滚动
- 每个区域头部显示图标 + 区域名 + 数量
- 照片卡片操作后实时移入对应区域，计数实时更新

### 3.3 照片卡片

保留区卡片较大(160px宽)，待定区和删除区卡片较小(100px宽)。

每张卡片显示：
- 缩略图
- 左上角状态标记（✓保留 / ?待定 / ✗删除）
- 右下角综合评分

选中态：红色边框 + 脉冲动画

### 3.4 详情面板

右侧固定 320px 宽，不包含图片预览区域。用户如需查看大图，**双击照片卡片调用系统默认图片查看器打开**。

详情面板内容从上到下：
1. 文件信息（缩略图 + 文件名 + EXIF 摘要）
2. 审美档案（核心亮点 + 四维度分数条）
3. 技术档案（各维度分数 + 修复建议）
4. 综合建议 + 操作按钮（P/X/U）
5. 标签

### 3.5 查看大图

**双击照片卡片 → 调用操作系统默认图片查看器打开**

- Windows: 调用 `start` 命令或 Shell API 打开
- 不在工具内实现图片放大功能，减少复杂度
- Z 键放大功能取消，用系统查看器替代

## 四、弹框交互设计

### 4.1 理由选择弹框

#### 触发逻辑

> **只在用户判断与系统判断不一致时弹出。一致时不弹。**

| 场景 | 是否弹理由 | 原因 |
|------|-----------|------|
| 系统建议保留 → 你保留 | ❌ 不弹 | 一致，无新信息 |
| 系统建议保留 → 你删除 | ✅ 弹删除理由 | 你否定了系统——高价值学习信号 |
| 系统建议淘汰 → 你删除 | ❌ 不弹 | 一致，无新信息 |
| 系统建议淘汰 → 你保留 | ✅ 弹保留理由 | 你否定了系统——高价值学习信号 |
| 系统不确定 → 你保留 | ✅ 弹保留理由 | 系统没判断，你提供了信号 |
| 系统不确定 → 你删除 | ✅ 弹删除理由 | 系统没判断，你提供了信号 |

#### 弹框结构

```
┌─────────────────────────────────────────┐
│  选择原因                                    ✕ │
│                                           │
│  🧠 当前为学习阶段，建议选择原因帮助我更懂  │
│  你的偏好。随着学习深入，此弹框出现频次会    │
│  逐渐降低。                                │
│                                           │
│  ○ 没有亮点，平平无奇                      │
│  ○ 背景杂乱，画面不干净                    │
│  ○ 表情/姿态不好                          │
│  ○ 构图不舒服                              │
│  ○ 光线/氛围不好看                        │
│  ○ 有更好的同类照片了                      │
│  ○ 其他                                   │
│    ┌─────────────────────────────────┐    │
│    │ 请输入具体原因... (必填)         │    │
│    └─────────────────────────────────┘    │
│                                           │
│           [确认]                           │
└─────────────────────────────────────────┘
```

#### 删除理由选项

| # | 选项 | 说明 |
|---|------|------|
| 1 | 没有亮点，平平无奇 | 无任何突出之处 |
| 2 | 背景杂乱，画面不干净 | 环境干扰主体 |
| 3 | 表情/姿态不好 | 人像场景常见 |
| 4 | 构图不舒服 | 画面布局问题 |
| 5 | 光线/氛围不好看 | 光影或色调不理想 |
| 6 | 有更好的同类照片了 | 相对比较后淘汰 |
| 7 | 其他（需填写） | 选择后弹出必填输入框 |

#### 保留理由选项

| # | 选项 | 说明 |
|---|------|------|
| 1 | 画面好看，光线美 | 视觉吸引力 |
| 2 | 表情/情绪好 | 情感共鸣 |
| 3 | 瞬间难得，错过不再有 | 稀缺性 |
| 4 | 有故事感 | 叙事性 |
| 5 | 整体感觉对 | 直觉判断 |
| 6 | 后期空间大 | 技术可修性 |
| 7 | 其他（需填写） | 选择后弹出必填输入框 |

#### 交互规则

- **单选**：最重要的那个理由最有学习价值
- **选项 1-6**：选中后可直接确认
- **选项 7"其他"**：选择后出现必填输入框，不填无法确认
- **可跳过**：右上角 × 关闭弹框，直接完成操作（跳过不影响操作本身）
- **确认后**：照片移入对应区域，理由记录到决策历史

#### 渐退机制

| 阶段 | 条件 | 行为 |
|------|------|------|
| 冷启动期 | 前 100-200 次操作 | 每次与系统不一致时弹出 |
| 校准期 | 维度权重稳定 + 预测准确率 > 70% | 仅在判断不一致时弹出（频次已降低） |
| 信任期 | 各场景 30+ 样本 + 高置信度 | 几乎不主动问，仅用户主动反馈时出现 |

## 五、快捷键体系

| 动作 | 快捷键 | 说明 |
|------|--------|------|
| 保留 | P | 移入保留区 |
| 删除 | X | 移入删除区（可能弹理由选择） |
| 待定 | U | 移入待定区 |
| 星标 1-5 | 1 2 3 4 5 | 写入 XMP，与 Lightroom 一致 |
| 去除星标 | 0 | 写入 XMP |
| 下一张 | → / 空格 | 浏览推进 |
| 上一张 | ← | 回看 |
| 下一组 | Tab | 相似组之间跳转 |
| 撤销 | Ctrl+Z | 撤销上一步，照片移回原区域 |
| 双击 | — | 调用系统图片查看器打开 |

**设计细节**：
- P/X/U 在左手区域，右手可留在鼠标上
- 星标和保留/删除独立：打星标 ≠ 保留，星标写 XMP，保留/删除是文件移动
- 撤销栈保留最近 50 步操作

## 六、虚拟滚动方案

800+ 张照片的展示性能关键点：

- 使用虚拟滚动库（如 vue-virtual-scroller），只渲染可视区域的卡片
- 缩略图按需加载：滚动到可视区域时才生成/加载缩略图
- 缩略图尺寸：保留区 160px 宽，待定/删除区 100px 宽
- 缩略图缓存：Rust 后端生成后存入临时目录，避免重复生成
